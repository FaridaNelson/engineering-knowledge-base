If you're reading this, it means you're determined to become a software engineer. If so, soon enough you'll embark on an exciting journey called "finding a job." The recruitment process always varies: while one company may have you go over your work experience and take a coding test, another organization might give you a technical interview.

You'll prepare for the first kind of interview more during the career prep course, but we'll start preparing you for a technical interview right now.

image

A technical interview usually consists of two parts. The first part includes questions concerning the JavaScript language, making a layout, and creating a website. The second part is all about real-world tasks, namely, writing code. From here on out, at the end of each sprint, we'll have a chapter dedicated to preparing for a technical interview. Some chapters will deal with advanced JavaScript paradigms and various browser nuances, while others will focus on writing code and solving tasks.

You're about to begin the first chapter in this series, where we'll take a look at object prototypes. Interviewers often ask questions about prototypes, and knowing how to work with prototypes is crucial for any JavaScript developer. Plus, understanding Prototypes helps us to see how JavaScript actually works.

You and JavaScript are about to become much closer. You have our word.
